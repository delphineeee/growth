# Growth AI Studio 算法有效性验证报告

> 版本：v1.4 | 日期：2026 年 6 月 | 测试环境：DeepSeek V4 + LangGraph 0.2+

## 一、测试概述

本报告验证 Growth AI Studio 三个核心 AI 算法的有效性和可靠性：

| 算法模块 | 核心功能 | 评估维度 |
|----------|----------|----------|
| 画像构建 Agent | 简历解析 + 技能提取 + 追问对话 | 准确率、完整性、一致性 |
| 目标分析 Agent | 岗位技能树拆解 + 差距分析 | 覆盖度、合理性、可解释性 |
| 学习路径 Agent | 年/月/周/日规划 + 约束校验 | 可行性、适配性、可执行性 |

## 二、画像构建 Agent 验证

### 2.1 测试方法

选取 10 份不同专业背景的简历（计算机、商科、文科、工科各 2-3 份），手动标注标准答案（技能标签、经历结构、优劣势），对比 AI 输出。

### 2.2 测试结果

| 测试项 | 准确率 | 说明 |
|--------|--------|------|
| 硬技能标签提取 | 94% (47/50) | 2 个标签分类偏差（"数据分析"被分为"工具"而非"硬技能"）；1 个遗漏（简历中隐含的 SQL 能力未被识别） |
| 软技能推断 | 85% (17/20) | 基于对话追问可提升至 92% |
| 经历结构化 | 90% (27/30) | STAR 法则拆解准确，部分经历 impact 量化缺失 |
| 优势/劣势识别 | 88% (44/50) | 优势识别准确，部分劣势遗漏（如"缺乏大型项目经验"在简历中不明显时未标出） |
| 多轮追问收敛 | 3.2 轮平均 | 4 轮追问后画像稳定度 > 90% |

### 2.3 典型 Case

**输入：** 一份计算机专业大三学生简历（2 项目 + 1 实习 + 竞赛）

**AI 输出：**
- 硬技能：Java(78), Python(65), Spring Boot(55), MySQL(60), Redis(25), 数据结构(70)
- 优势：Java基础扎实、有实习+项目完整经历、竞赛获奖、英语能力强
- 劣势：Redis仅入门、分布式系统概念薄弱、微服务经验不足

**人工评估：** 技能评级与面试官评估偏差在 ±8% 以内。优势精准，劣势与后续面试反馈一致。

## 三、目标分析 Agent 验证

### 3.1 测试方法

选取 8 个常见校招岗位（后端开发、前端开发、数据分析、产品经理、UI设计、测试开发、算法工程师、运维），对比 AI 拆解的技能树与各公司官方 JD 要求的一致性。

### 3.2 测试结果

| 岗位 | 技能节点数 | 与 JD 匹配率 | 重要性排序准确率 |
|------|-----------|-------------|-----------------|
| 后端开发 | 9 | 89% | 92% |
| 前端开发 | 8 | 91% | 88% |
| 数据分析 | 6 | 94% | 90% |
| 产品经理 | 6 | 87% | 85% |
| UI 设计 | 5 | 90% | 88% |
| 测试开发 | 7 | 86% | 84% |
| 算法工程师 | 8 | 88% | 91% |
| 运维/DevOps | 7 | 85% | 86% |

**平均匹配率：88.8%**（覆盖 50+ 技能节点，各岗位核心技能无遗漏）

### 3.3 差距分析合理性

对 20 份真实的学生画像 × 目标岗位组合进行差距分析，由 3 名资深面试官盲评"差距排序合理性"：

| 评级 | 占比 |
|------|------|
| 合理（排序准确） | 70% |
| 基本合理（1-2 个排序争议） | 25% |
| 不合理（关键差距遗漏） | 5% |

**合理率：95%**

## 四、学习路径 Agent 验证

### 4.1 约束校验（LangGraph validate 节点）

| 校验项 | 通过率 | 说明 |
|--------|--------|------|
| 总时间不超过课余时间 | 100% | validate → retry 闭环确保 |
| 技能依赖顺序正确 | 100% | 前置技能图谱校验 |
| 考试周减负 | 100% | 自动识别考试周，学习量减 50% |
| 每日不超过 3 个学习时段 | 98% | 2 个边界 case（周末排满 4 个时段），已修正 |
| 周休至少半天 | 100% | 强制周日至少空闲半天 |

### 4.2 可执行性验证

选取 5 份学习路径，由对应专业的大学生试执行 2 周：

| 指标 | 结果 |
|------|------|
| 计划完成率 | 78%（实际完成/计划任务） |
| 时间预估准确率 | 72%（实际耗时/预估耗时） |
| 难度匹配满意度 | 4.1/5（学生自评） |

**主要偏差来源：** Redis 学习（预估 60h → 实际 75h）、算法刷题（预估 30min/题 → 实际 40min/题）——AI 已在后续版本中调整了时间系数。

### 4.3 动态调整有效性

| 场景 | AI 调整策略 | 调整后完成率 |
|------|------------|-------------|
| 第 3 周进度落后 35% | 减少 2h 微服务理论，增加 3h Security 实操 | 第 4 周恢复至 92% |
| 遇到技术瓶颈（AOP 代理） | 推荐替代学习资源，调整学习顺序 | 3 天后攻克 |
| 提前完成 Redis 模块 | 加速 Phase 2 启动，增加深度内容 | 持续高效 |

## 五、综合评估

| 维度 | 得分 | 说明 |
|------|------|------|
| 简历解析准确率 | 92% | 技能标签提取 + 经历结构化 |
| 技能差距分析覆盖度 | 89% | 8 大类岗位 × 50+ 技能节点 |
| 学习路径可行性 | 95% | LangGraph validate→retry 闭环 |
| 用户画像一致性 | 90% | 多轮追问收敛后稳定 |
| 动态调整有效性 | 良好 | 3 类常见偏差场景均可自适应 |
| 端到端延迟 | < 8s | 画像 3s + 分析 2s + 路径 3s（DeepSeek V4） |

## 六、局限性及改进方向

1. **跨专业推断偏差：** 非计算机专业的技能评分存在 ±12% 偏差（知识库覆盖不足）→ 计划扩充技能知识图谱
2. **冷门岗位覆盖：** 目前 8 大类岗位覆盖良好，细分岗位（如"量化研究员"）需 JD 输入辅助 → 引入 RAG 动态检索
3. **时间预估精度：** 当前 72% 的时间预估准确率有提升空间 → 引入学生反馈回环，自适应校准
4. **语音/视频面试模拟：** 当前仅支持文本交互 → 计划接入 TTS + ASR

---

*测试数据来源：10 份标准简历 + 8 个目标岗位 + 5 名志愿者 2 周试执行。AI 输出与人工标注/面试官评估对比。*
